# 授权绕过

## 1.定义：

**授权绕过（Authorization Bypass）** 是 Web 安全中最常见、危害极大的漏洞（OWASP Top 10 2021 首位：失效的访问控制），指攻击者**跳过或破坏系统的权限校验机制**，在未登录或权限不足的情况下，非法访问、修改敏感数据或执行高权限操作。

## 3.DVWA的中级防御

1.防御：在低级防御的基础上

```php
<?php
/*

Only the admin user is allowed to access this page.

Have a look at these two files for possible vulnerabilities: 

* vulnerabilities/authbypass/get_user_data.php
* vulnerabilities/authbypass/change_user_details.php

*/

if (dvwaCurrentUser() != "admin") {
    print "Unauthorised";
    http_response_code(403);
    exit;
}
?>
```

**中级防御 = 给 “主页面” 加了权限判断，但没给 “真实功能接口” 加权限！**

DVWA 在 Medium 安全级别，只做了这一件事：

1. **访问 `/vulnerabilities/authbypass/` 主页面时**

   后端会检查：**你是不是管理员？**

   不是 → 直接拦截，返回 **403 未授权**。

2.绕过：

虽然给 “主页面” 加了权限判断，但没给 “真实功能接口” 加权限！

所有直接访问接口：

```url
http://192.168.144.1/dvwa/vulnerabilities/authbypass/get_user_data.php
```

## 3.DVWA的高级防御

1.防御：高级防御，页面隐藏、页面限制、接口限制，但限制都是前端，（后端只限制是不是本网站的请求头）我直接抓包提交合法请求就能绕过

```php
<?php
/*

Only the admin user is allowed to access this page.

Have a look at this file for possible vulnerabilities: 

* vulnerabilities/authbypass/change_user_details.php

*/

if (dvwaCurrentUser() != "admin") {
    print "Unauthorised";
    http_response_code(403);
    exit;
}
?>
```

2.绕过;

BP直接抓包：

把GET包改成POST包

在下面直接添加一个有权限的用户

```php
{
  "id": 5,
  "first_name": "cao",
  "surname": "licheng"
}
```

#### 为什么可以直接添加管理员用户

你是利用了**接口的逻辑缺陷**

这个接口的代码设计是：

> “只要你是本站登录用户，你就可以修改任何用户的资料。”

它没有做：

当前用户 (gordonb) 是否有权限修改 ID 为 5 的用户？

因为它没做这个校验，所以你可以任意修改。
