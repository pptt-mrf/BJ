# 1.蛮力破解

1.定义

蛮力破解（也叫暴力破解）就是**用程序自动化地尝试所有可能的密码 / 密钥组合**，直到找到正确的那一个 —— 本质是 “穷举试错”，靠 “量大 + 速度快” 突破登录 / 加密防护。

2.分类

| 类型           | 特点                                             | 适用场景                     |
| -------------- | ------------------------------------------------ | ---------------------------- |
| **纯暴力穷举** | 遍历所有字符组合（如 a-z、0-9），最 “笨” 但全面  | 短位数、简单字符集的密码     |
| **字典攻击**   | 用包含常用密码、单词、个人信息的 “字典” 批量尝试 | 现实场景（人类设置的弱密码） |
| **密码喷洒**   | 用少量通用密码批量攻击大量账户，避免被锁定       | 大规模系统批量入侵           |
| **彩虹表攻击** | 预计算哈希值的高效字典攻击，专门破解哈希值       | 破解存储的密码哈希           |

3.扩展

**暴力破解不止能猜密码，还能用来「扫网页的隐藏文件 / 目录」**，比如找后台、找配置文件，工具会自动发请求、分析响应，只是不同工具的判断方式有优劣。

**4.运行python BruteForceAI.py attack --urls urls.txt --usernames usernames.txt --passwords passwords.txt --mode passwordspray --threads 3命令；**（重点）

| 命令片段                    | 核心作用       | 通俗解释                                                     |
| --------------------------- | -------------- | ------------------------------------------------------------ |
| `python`                    | 执行入口       | 调用 Python 解释器来运行后面的脚本（因为 BruteForceAI 是 Python 写的） |
| `BruteForceAI.py`           | 脚本主体       | 你要运行的爆破工具主程序文件                                 |
| `attack`                    | 核心指令       | 告诉工具：“我要执行「登录爆破」这个核心功能”（工具支持 analyze/attack 等多个功能，attack 是爆破） |
| `--urls urls.txt`           | 指定目标       | 工具会读取`urls.txt`里的网址（比如 DVWA 的登录页http://127.0.0.1/dvwa/login.php），只对这些网址进行爆破 |
| `--usernames usernames.txt` | 指定用户名列表 | 读取`usernames.txt`里的所有用户名（比如 admin、test、user），作为爆破的 “账号候选” |
| `--passwords passwords.txt` | 指定密码列表   | 读取`passwords.txt`里的所有密码（比如 123456、password123），作为爆破的 “密码候选” |
| `--mode passwordspray`      | 爆破模式       | 这是**核心优化参数**，决定爆破的策略：❌ 低效模式（默认）：一个用户名试完所有密码，再换下一个（比如 admin 试 200 个密码，再试 test）✅ passwordspray（密码喷洒）：一个密码试完所有用户名，再换下一个（比如 123456 试 8 个用户名，再试 password123）👉 优势：减少单账号被锁风险，靶场中弱密码通用时命中率更高 |
| `--threads 3`               | 并发数         | 同时开 3 个 “线程”（可以理解为 3 个 “工人”）并行测试，线程数越高，爆破速度越快（但太高会被靶场拦截） |

（1）核心指令

| 指令名          | 核心作用         | 通俗解释                                                     | 典型用法示例                                                 |
| --------------- | ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `attack`        | 登录爆破（核心） | 执行用户名 + 密码的爆破，尝试破解登录密码                    | `python BruteForceAI.py attack --urls urls.txt --usernames usernames.txt --passwords passwords.txt` |
| `analyze`       | 登录表单分析     | 自动识别目标网址的登录表单（比如用户名 / 密码输入框的名称、提交方式），为爆破做准备 | `python BruteForceAI.py analyze --urls urls.txt --llm-model llama3.2:3b` |
| `clean-db`      | 清理数据库       | 删除工具记录的 “已测试组合”“表单分析结果” 等缓存数据，解决 “跳过已尝试密码” 的问题 | `python BruteForceAI.py clean-db`                            |
| `check-updates` | 检查更新         | 检测工具是否有新版本，避免因版本老旧导致功能异常             | `python BruteForceAI.py check-updates`                       |

（2）爆破模式

| 模式名称 | 英文参数（--mode）  | 核心逻辑                                 | 通俗解释                                                     | 适用场景                                                    |
| -------- | ------------------- | ---------------------------------------- | ------------------------------------------------------------ | ----------------------------------------------------------- |
| 密码喷洒 | `passwordspray`     | 一个密码试完所有用户名，再换下一个密码   | 比如：先用`123456`试`admin/test/user`，再用`password123`试所有用户名 | ✅ CTF 靶场 / 测试环境（弱密码通用）✅ 避免单账号多次输错被锁 |
| 暴力枚举 | `brute`（默认模式） | 一个用户名试完所有密码，再换下一个用户名 | 比如：先用`admin`试`123456/password123/abc123`，再用`test`试所有密码 | ❌ 效率低，不推荐（除非目标是单账号）⚠️ 易触发账号锁定        |
| 组合爆破 | `combination`       | 按 “用户名 + 密码” 自定义组合规则测试    | 比如：只试`admin:admin`、`test:test`、`user:user`这类同名组合 | ✅ 已知目标有 “账号密码相同” 规律时                          |
| 增量爆破 | `incremental`       | 按密码长度 / 字符集增量生成密码测试      | 比如：先试 6 位数字（123456），再试 7 位（1234567），逐步增加 | ❌ 仅适合无字典的纯暴力破解（耗时极长）                      |
