# 现实工作中找 Web 漏洞的标准步骤（通用版）

## 1. 信息收集（最重要，占 60% 工作量）

不收集信息，你永远找不到漏洞。

重点看这些：

- 域名、子域名（一个主站可能有几十个弱鸡子站）

- 旁站、C 段网站

- 网站用的技术：

  - 语言：PHP / Java / ASPX / Node.js
  - 中间件：Apache / Nginx / Tomcat / IIS
  - 框架：ThinkPHP、SpringBoot、Struts2 等
  - CMS：织梦、帝国、WordPress、Discuz

  

- 真实路径、备份文件、git 泄露、.env 泄露

- 端口：21、22、80、443、8080、3306、3389、7001、8000~9000 等

- 邮箱、手机号、历史漏洞、历史密码

**现实规律：**

90% 的漏洞不是在首页，而是在 ** forgotten 页面、测试页面、旧系统、子站、备份 ** 里。

------

## 2. 目录扫描 & 接口探测

工具：

- dirsearch
- dirmap
- burp 爬虫
- 手工点一遍全站

找：

- 后台 /admin/manage /system
- 上传接口 /upload/file
- 测试页面 /test/phpinfo.php
- 接口文档 /swagger/api /v2/api-docs
- 备份文件：www.rar、web.zip、sql.bak、.git、.svn、.env

**现实里最容易出漏洞的就是：旧接口、遗留文件、未授权访问。**

------

## 3. 抓包，看所有请求与参数（核心）

打开 Burp Suite 开抓，点遍整个网站。

重点关注：

- URL 里的参数：`?id=` `?uid=` `?page=` `?file=`
- POST 提交：登录、查询、修改资料、发布内容
- 上传文件
- Cookie、Referer、User-Agent
- 接口参数：json 里的 id、userid、orderid

**现实里 70% 漏洞都在参数里。**

------

## 4. 按漏洞类型挨个测试（工作标准顺序）

### ① 未授权访问（最简单、最常见）

测试：

- 直接访问后台
- 直接访问用户信息接口
- 直接访问管理员功能
- 直接访问数据库备份

很多站**根本没登录校验**。

### ② 越权漏洞（水平越权 / 垂直越权）

现实中**最多、最容易拿高分**的漏洞。

测试方法：

- 用 A 用户的 ID 去看 B 用户信息
- 改 userid、uid、id、orderid
- 普通用户尝试访问管理员接口

越权是 SRC 奖金常客。

### ③ SQL 注入

测试点：

- 所有数字参数
- 搜索框
- 登录框
- 订单号、用户 ID

简单测试：

plaintext











```
id=1'
id=1 and 1=1
id=1 and 1=2
```

### ④ XSS 跨站脚本

测试点：

- 昵称、个性签名
- 搜索框
- 留言、评论
- 任何你能输入并显示的地方

payload：

plaintext











```
'><script>alert(1)</script>
```

### ⑤ 文件上传

找上传点：头像、文章图片、证书上传等。

尝试：

- 直接上传 php
- 改后缀：php.jpg、php%00.jpg
- 双写后缀：pphphp
- 图片马

### ⑥ 命令执行 / 代码执行

多见于：

- ping 功能
- 备份、解压
- 模板编辑
- 特定框架漏洞（ThinkPHP 某些版本）

### ⑦ 文件包含 / 目录遍历

URL 类似：

plaintext











```
?file=
?page=
?path=
```

测试：

plaintext











```
../../etc/passwd
../../windows/system32/drivers/etc/hosts
```

### ⑧ 弱口令 & 爆破

后台、FTP、邮箱、SSH

账号密码组合：

admin/admin、admin/123456、root/root、公司域名相关

------

## 5. 框架 / 中间件漏洞（实战神器）

看到以下直接查 POC：

- Tomcat 弱口令 + 后台部署 war
- JBoss、WebLogic 未授权访问
- Nginx 目录遍历
- Apache 任意文件读取
- ThinkPHP 5/3 代码执行
- SpringBoot 未授权访问
- Shiro 反序列化
- Fastjson 反序列化

**现实中很多站就是死在框架漏洞。**

------

## 6. 手工验证 + 绕过 WAF

真实环境都有 WAF，不能直接 sqlmap 梭哈。

一般做法：

- 分块注入
- 大小写绕过
- 编码绕过
- 参数污染
- 换 UA、加 Referer
- 延时注入、布尔注入

------

## 7. 漏洞确认 → 复现 → 写报告

工作里不是拿到 shell 就完事，而是：

- 复现漏洞
- 截图
- 写复现步骤
- 给出危害
- 给出修复方案

------

# 最真实的一句话总结（工作版）

**先收集信息 → 扫目录找入口 → 抓包看参数 → 优先测未授权 / 越权 → 再测注入 / XSS / 上传 → 最后看框架漏洞**

现实工作中：

- 越权漏洞最多
- 未授权访问第二
- SQL 注入第三
- 文件上传第四
- 弱口令永远有效

# 第一步，信息收集

好，我直接给你一份**真正工作中、最完整、最实用的信息收集清单**。

你以后打任何站，**照着这条清单收集一遍，漏洞基本就自己跑出来了**。

不废话、不玄学，全是一线渗透必收的内容。

------

## 一、域名 / 站点基础信息（最外层）

1. **主域名、子域名**

   

   - 比如：`www.xxx.com`、`admin.xxx.com`、`api.xxx.com`、`test.xxx.com`
   - 子域名经常比主站弱得多，是突破口

   

2. **旁站 / C 段**

   

   - 同一个服务器上的其他网站
   - 同一个 IP 段的其他机器（C 段：192.168.1.x）

   

3. **备案信息、公司信息**

   

   - 公司名、法人、电话、邮箱

     

     → 用来猜弱口令、后台路径、社工密码

   

4. **历史解析记录（DNS 历史）**

   

   - 可能遗留旧服务器、测试站、未删的漏洞站

   

------

## 二、网站指纹（判断用什么语言、框架、CMS）

这是**最关键**的，决定你后面用什么漏洞。

你要收集：

1. **开发语言**

   

   - PHP、Java、ASPX、ASP、Node.js、Python

     

     → 看 URL 后缀：php /jsp/asp /aspx/do /action

   

2. **中间件（服务器）**

   

   - Nginx、Apache、Tomcat、IIS、WebLogic、JBoss、Jetty

   

3. **数据库**

   

   - MySQL、SQLServer、Oracle、PostgreSQL

   

4. **CMS 系统（占实战 60%）**

   

   - 织梦 DedeCMS

   - 帝国 CMS

   - WordPress

   - Discuz

   - PHPCMS

   - 齐博、方维、ThinkPHP 自带后台

     

     → 每个 CMS 都有固定漏洞、固定后台路径

   

5. **开发框架**

   

   - ThinkPHP 5 / 3

   - SpringBoot、SpringMVC

   - Struts2

   - Laravel

     

     → 框架漏洞基本都是

     一键拿 shell

   

6. **前端指纹**

   

   - jQuery、Vue、React、Layui

     

     不重要，但能辅助判断系统新旧

   

------

## 三、端口信息（打开哪些服务）

必扫端口：

- 21 – FTP（经常弱口令）
- 22 – SSH（Linux 远程）
- 80 / 443 – Web
- 3306 – MySQL
- 1433 – SQL Server
- 8080 – Tomcat
- 7001 / 7002 – WebLogic
- 8080–9090 一堆后台
- 3389 – Windows 远程桌面

端口开得多，**漏洞概率成倍增加**。

------

## 四、目录、文件、隐藏路径（漏洞重灾区）

这是新手最容易忽略，但**实战最容易出漏洞**的地方。

要收集：

1. **robots.txt**

   直接告诉你哪些目录不让搜索引擎看，往往就是后台。

   

2. **常见后台**

   

   - /admin
   - /manage
   - /system
   - /login
   - /admin_login
   - /backend

   

3. **接口目录**

   

   - /api
   - /api/v1
   - /swagger
   - /v2/api-docs
   - /doc.html

   

4. **上传目录**

   

   - /upload
   - /upfile
   - /file
   - /images

   

5. **敏感文件 / 备份文件**

   

   - www.rar
   - web.zip
   - backup.sql
   - db.sql
   - .git （Git 泄露，能下整个源码）
   - .svn
   - .env （Laravel 数据库密码直接明文）
   - config.ini
   - web.config
   - phpinfo.php
   - test.php

   

6. **安装 / 卸载 / 重装页面**

   

   - /install

   - /setup

   - /install.php

     

     能进这些页面，

     直接重建管理员账号拿站

   

------

## 五、源码泄露信息（看页面源代码）

右键查看源代码，找：

1. 注释里的路径、账号、密码、IP
2. JS 里的接口路径、API key、token
3. 隐藏表单域：`userid=1`、`role=admin`
4. 旧链接、测试接口、测试功能

很多站源码里直接写着：

```
// 测试账号 admin / admin123
// 数据库密码 root@123
```

------

## 六、用户信息、账号体系

1. 有没有**注册功能**

2. 有没有**找回密码**（短信轰炸、验证码绕过、越权改密码）

3. 用户名规则：手机号、身份证、邮箱、固定前缀

4. 管理员账号常见规律：

   - admin
   - root
   - system
   - 公司拼音
   - 法人姓名

   

------

## 七、业务功能点（决定漏洞类型）

记录网站有哪些功能：

- 登录
- 注册
- 搜索
- 上传头像 / 附件
- 留言 / 评论
- 订单 / 支付
- 管理员后台
- 文件管理
- 数据导出
- 数据库备份
- Ping、Tracert 网络测试功能

每个功能对应一类漏洞：

- 上传 → 文件上传
- 搜索 → SQL 注入 / XSS
- 订单 / 用户 ID → 越权
- Ping → 命令执行
- 备份 → 任意文件下载

------

## 八、WAF / 防护信息（决定你能不能打得动）

收集：

- 是否有 WAF（阿里云、腾讯云、安全狗、云盾、长亭）
- 触发拦截的关键词
- 是否屏蔽扫描器
- 是否限制请求频率

------

## 超级浓缩一句话版（你记这个就行）

信息收集 = 收集**域名 + IP + 端口 + 语言 + 中间件 + 框架 + CMS + 后台路径 + 上传接口 + 备份文件 + 源码泄露 + 业务功能 + WAF**

收集完这些，你根本不用 “找漏洞”，

漏洞会自己摆在你面前。
