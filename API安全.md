# API安全

#### 1.定义：

**API = 应用程序编程接口**

你可以把它理解成：**软件之间的 “翻译官” 或 “服务员”**，负责让不同程序互相沟通、调用功能。

#### 2.Web API 是什么

**Web API = 跑在网页 / 服务器上的 “接口”**

它的作用就是：

让**两个不同的软件**，通过网络（HTTP/HTTPS）互相说话、传数据、调用功能。

#### 3.为什么现在 API 这么重要

云计算、微服务架构和单页应用等不同技术的采用，都推动了 API 作为架构运动的普及。

#### 4.API 安全为什么致命

与引入任何新概念一样，可能存在缺陷和漏洞，需要测试。否则，安全不足的 API 可能会提供对敏感数据的无限制直接路径。

API 不像传统网页有界面遮挡，它**直接暴露逻辑**：

如果 API 没做安全控制：

攻击者不需要破解后台，

## 2.分类

### 1. REST API（最常见、测试最多）

全称：**Representational State Transfer**（表示状态传输）

现在互联网 90% 都是这种。

特点：

- 基于 HTTP/HTTPS
- 用标准方法：GET、POST、PUT、DELETE
- 数据格式：JSON 为主
- 地址像网址：`/api/user/123`
- 无状态，靠 Token / Cookie 鉴权

安全测试重点：

- 未授权访问
- IDOR（直接改 ID 看别人数据）
- 水平 / 垂直越权
- 参数注入、敏感信息泄露

一句话：**你做渗透测试，遇到最多的就是 REST API。**

------

### 2. SOAP API（老、重、企业 / 银行常用）

全称：**Simple Object Access Protocol**（简单对象访问协议）

特点：

- 基于 XML，格式严格复杂
- 有严格规范（WS-Security）
- 多用于银行、政务、旧企业系统
- 通常走 POST，内容一大段 XML

安全测试重点：

- XML 注入
- XML 外部实体注入（XXE）
- 弱 WS-Security、认证绕过

一句话：**老系统、金融系统常见，漏洞偏 XML 类型。**

------

### 3. GraphQL API（灵活、容易信息泄露）

一种**查询语言**，不是传统接口。

特点：

- 客户端自己决定要哪些字段
- 一条查询可以拉很多数据
- 格式固定：`POST /graphql`
- 有 schema 结构

安全测试重点：

- 批量查询导致拖库
- 越权访问任意对象
- 敏感字段未加权限控制
- 暴力遍历 ID

一句话：**功能越强，越容易被一次性扒光数据。**

------

### 4. gRPC（高性能、微服务内部常用）

全称：**Google Remote Procedure Call**

特点：

- 基于 HTTP/2
- 用 Protobuf 二进制格式（不是明文）
- 速度快，多用于微服务之间通信
- 不像 REST 那样好抓包看懂

安全测试重点：

- 未授权访问
- 服务间权限绕过
- 越权调用方法
- 流量加密不足

一句话：**微服务架构必见，抓包看不懂，需要工具解析。**

------

### 5. WebSockets（实时通信、长连接）

不是传统请求 - 响应，而是**持续连接**。

特点：

- 用于聊天、直播弹幕、实时通知
- 协议：ws:// 或 wss://
- 客户端和服务器互相主动发消息

安全测试重点：

- 越权发送消息
- 身份伪造
- 消息注入
- XSS、信息泄露

一句话：**实时交互的地方基本都是 WebSocket。**

## 3.REST

1.定义：REST 不是一种技术，不是语言，而是一套 “写 API 的规矩”。

2.REST API 的核心就是：**用地址定位资源**

3.**REST 是一套写 API 的规则**

用 **URI 定位资源**

用 **HTTP 方法（GET/POST/PUT/DELETE）做增删改查**

用 **Header 传身份和格式**

用 **状态码表示结果**

因为简单通用，成为当今最主流的 API 架构

## 4.OAuth 2.0（规则）

1.定义：让第三方应用在**不拿到你账号密码**的前提下（获得一个Token ），有限度地访问你的数据 / 功能。

 token：是一种临时房卡一样的东西啊？只不过只能用一次，而且权限有限

2.安全视角（和你打靶场相关）

- Token 是临时的，过期就失效
- 攻击者如果偷到你的 Token，就能冒充你访问 API（**会话劫持**）
- 测试重点：看 Token 会不会泄露、能不能被篡改、权限有没有控制好

## 5. Postman 

1. 核心作用（3 句话）
   发请求：像浏览器一样给 API 地址发 GET/POST/PUT/DELETE 请求，还能自定义 Header、Body、参数
   看结果：直接显示服务器返回的 JSON/XML 数据、状态码、响应时间
   自动化：保存常用请求、批量测试、自动获取 OAuth 2.0 Token（比如你 DVWA 里的 API 就用它来拿 Token）

2. 一句话总结

**Postman 就是 API 测试的「瑞士军刀」，专门用来发请求、看数据、测漏洞，是你打 DVWA API 模块的核心工具。**

## 6.Burp Suite

1.核心用「Repeater（重放器）」模块：抓包 / 手动构造 API 请求，改 OAuth2.0 的 Token、请求方法（GET/POST）、参数，直接测越权 / 未授权访问，**比 Postman 更适配渗透测试**（毕竟是专门的漏洞挖掘工具）
